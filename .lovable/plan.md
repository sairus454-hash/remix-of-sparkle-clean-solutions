## Проблемы из аудита и план решения

### 1. Дубликаты контента (100% совпадение всех страниц)
**Причина:** SEO-краулер (saitreport.ru) не выполняет JavaScript и видит одинаковый HTML-шаблон (index.html) для всех страниц. Google умеет рендерить JS, но сторонние краулеры — нет.

**Решение A — SSR-prerender через Vercel Middleware:**
- Обновить `seo-prerender` edge-функцию в Supabase, чтобы она генерировала уникальный HTML с мета-тегами для каждого URL
- НО: Vercel middleware на Lovable hosting не поддерживается напрямую. Альтернатива — добавить мета-теги прямо в `index.html` через Vite-плагин (SSG-подход)

**Решение B — Уникализация контента городов:**
- Добавить уникальные описания, тексты, FAQ для каждого города в `cities.ts`
- Добавить блок уникального текста на CityPage

### 2. Мета-теги — дубли и пустые
- Проверить все 40+ страниц (города, блог, основные)
- Убедиться что каждая страница имеет уникальный title и description

### 3. Юзабилити — убрать лишние градиенты
- Упростить промо-баннер в Layout (слишком яркий)
- Убрать/смягчить градиенты на карточках услуг
- Улучшить контраст текста на цветных фонах
- Проверить отступы между секциями

### Порядок работы:
1. Начать с градиентов (быстрый визуальный эффект)
2. Уникализировать контент городов
3. Уникализировать мета-теги городов и блога
